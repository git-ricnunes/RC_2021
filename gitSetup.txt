Mini guia para setup Github

1. Gerar chave publica a adicionar ao Github:
	Na terminal linux correr os seguintes comandos com o respectivo email associado a conta de Github:
	ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
	
	Quando aparecer a seguinte mensagem pressionar enter (assumir o caminho default para as chaves publicas):
	Enter a file in which to save the key (/c/Users/you/.ssh/id_rsa):
	
	Definir uma password para a chave publica:
	Enter passphrase (empty for no passphrase): [Type a passphrase]
	Enter same passphrase again: [Type passphrase again]
	
2. Adicionar a chave ao ssh-agent do linux
	Correr o seguinte comando para confirmar se o ssh-agent está a funcionar:
	eval $(ssh-agent -s)
	
	Adicionar a chave publica gerada anteriormente ao shh-agent:
	ssh-add ~/.ssh/id_rsa
	
3. Adicionar a chave ao Github:
	Na terminal correr o seguinte comando para copiar o conteudo da chave para o clipboard:
	clip < ~/.ssh/id_rsa.pub
	
	Aceder ao site do Github, fazer login na conta e navegar até Settings->SSH and GPG keys
	Carregar no botão New SSH Key.
	Dar um nome a chave e colar o conteudo do clipboard na caixa de texto.
	Carregar em Add SSH key.
	
4. Testar a comunicação 
	Na terminal testar sacar o projeto de RC atraves da terminal com o comando:
	git clone git@github.com:git-ricnunes/RC_2021.git
	
	Colocar a password da conta Github.
	
5. Comandos basicos do git:
	git clone <nome do repositorio.git> --> faz uma copia local de um repositorio remoto.
	git pull --> actualiza o repositorio local do git com novo conteudo vindo do repositorio remoto.  
	git add <nome do ficheiro> --> Adiciona o ficheiro a zona de stagging
	git add -A --> Adicionada todos os novos ficheiros ou ficheiros alterados para a zona de stagging
	git restore <nome do ficheiro> --> restaura o ficheiro <nome do ficheiro> ao estado inicial (isto quer dizer que o ficheiro original sem alteração é reposto) 
	git status --> Verficar o estado do repositorio git , o que inclui ficheiros alterados que se encontram na zona de stagging ou não.
	git commit -m "<mensagem do commit>" --> Confirmar que os ficheiros em stagging são para avançar. De maneira a concluir é necessario colocar uma mensagem de commit <mensagem do commit>
	git push --> efetua o envio das alterações para o repositorio git.
	
	...
	Comandos que nao devemos utilizar no ambito do RC mas vai dar jeito para as cadeiras de SD e ES:
	git branch --> mostra todos os branches do repositorio.
	git checkout -b <nome do branch> --> cria uma nova branch com o nome <nome da branch> a partir da branch atual. 
	git merge <nome da branch> --> actualiza a branch atual com a branch <nome da branch>.
	
	
6. Dicas:
	Antes de qualquer alteração no codigo ou commit é muito importante fazer git pull. Se não for feito pode causar conflitos caso mais do que uma pessoa esteja a alterar o mesmo ficheiro. 
	Normalmente os conflitos de git mal geridos dão grandes dores de cabeça.
